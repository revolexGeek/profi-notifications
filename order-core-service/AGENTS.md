# lmm — правила для агента

Python-сервис управляет жизненным циклом заявок с Profi.ru, выполняет дедупликацию, хранит статусы обработки, запускает LLM-анализ и публикует команды на уведомление в RabbitMQ. Разрабатывается строго по Clean Architecture.

## Активируй скиллы перед работой

Перед любым кодом обязательно читай:
- `skill://clean-architecture` — всегда.
- `skill://tdd-workflow` — всегда: любой код пишется через TDD.
- `skill://docker-patterns` — при работе с Docker / compose.
- `skill://faststream` — при работе с rabbitmq и faststream
- `skill://pydantic-models-py` — при работе со схемами
- `skill://alembic` — при работе с миграциями

## Порядок работы (TDD обязателен)

Любую задачу выполняй строго в этом порядке:
1. Изучи кодовую базу: пойми затронутый слой, существующие порты, стиль и тесты рядом.
2. Активируй `skill://tdd-workflow` и следуй ему.
3. Работай циклом red-green-refactor: сначала падающий тест на нужное поведение, потом минимальный код чтобы он позеленел, потом рефактор без смены поведения.
4. Никакого продакшн-кода без предварительно написанного падающего теста.
